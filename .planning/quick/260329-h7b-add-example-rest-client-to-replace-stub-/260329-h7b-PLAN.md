## Objective

Add an example REST client implementation to replace the stub `CustomerCoreGateway`, demonstrating how engineers can integrate real downstream HTTP services.

## Implementation Plan

### Task 1: Add Quarkus REST Client dependency and create client interface

**Files:** `pom.xml`, `src/main/java/com/example/api/downstream/rest/CustomerCoreRestClient.java`

**Action:**
1. Add `quarkus-rest-client-jackson` dependency to pom.xml (if not present)
2. Create `CustomerCoreRestClient.java` interface with `@RegisterRestClient`
3. Define methods matching `CustomerCoreGateway` interface:
   - `getCustomerProfile(String customerId)`
   - `createCustomerProfile(CreateCustomerProfileRequest request)`
   - `updateCustomerProfile(String customerId, UpdateCustomerProfileRequest request)`
4. Add proper JAX-RS annotations (`@GET`, `@POST`, `@PUT`, `@Path`, `@Consumes`, `@Produces`)

**Verify:**
- Maven compiles without errors
- Client interface has correct REST annotations
- Methods return appropriate types (`CustomerProfileView`, `Response`, etc.)

**Done:** Client interface created and compiles

### Task 2: Create REST client implementation of CustomerCoreGateway

**Files:** `src/main/java/com/example/api/downstream/rest/RestCustomerCoreGateway.java`

**Action:**
1. Update `RestCustomerCoreGateway` to implement `CustomerCoreGateway`
2. Inject `CustomerCoreRestClient` using `@RestClient` annotation
3. Delegate gateway methods to REST client calls
4. Add configuration property for base URL (e.g., `customer.core.base.url`)
5. Handle exceptions and map to application exceptions

**Verify:**
- Gateway compiles and injects REST client correctly
- All `CustomerCoreGateway` methods are implemented
- Configuration property is documented

**Done:** REST client gateway implementation complete
